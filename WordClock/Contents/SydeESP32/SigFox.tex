\chapter{SigFox Connection Check}
Sigfox is a low-power wide-area network (LPWAN) technology designed for Internet of Things (IoT) applications. It enables devices to send small amounts of data over long distances while consuming minimal energy, making it ideal for battery-powered sensors and remote monitoring solutions. Unlike traditional cellular networks, Sigfox operates on unlicensed radio frequencies and uses a simple and efficient communication protocol, reducing costs and complexity for IoT deployments.

The Hypatia Board includes a Sigfox module, allowing it to communicate over this network. To verify that the Sigfox element is working correctly, we will create an example code that demonstrates how to use it. This example will include sending and receiving data via Sigfox and checking whether the device successfully connects to the network. Through this test, we can ensure that the Sigfox functionality on the Hypatia Board is properly configured and operational.

\section{Connecting Hypatia with Sigfox}

To set up and program the Hypatia Board, the first step is to install the Arduino IDE on your computer. Additionally, it is essential to verify that all required configurations for the Hypatia Board have been properly set up before proceeding.

Once the board is configured correctly and the appropriate Board and COM Port have been selected, the provided code must be compiled and uploaded to the ESP32 on the Hypatia Board. This process can be completed by clicking the upload button (see Figure \ref{fig:ArduinoUpload2}) located in the upper left corner of the Arduino IDE.

Once the upload is completed successfully, the Serial Monitor should be opened to monitor the board’s output and debug any potential issues. This can be done by selecting the corresponding button in the upper right corner of the Arduino IDE, as shown in Figure (see Figure \ref{fig:ArduinoSerialMonitor3}). Next step is to reassure that the baudrate, which can be adjusted in the upper right corner of the serial Monitor, is set to 9600 as it is initialized in the previous code.

The Serial Monitor allows real-time monitoring of the Hypatia Board's connection status to the SigFox network. Additionally, the Serial Monitor provides valuable insights into the Sigfox communication process, offering real-time feedback on the status and responses of the Hypatia Board's Sigfox module (see Figure \ref{fig:SigFoxSerialMonitor}). It displays the initialization process, including setting the Sigfox mode (LMS100A) and verifying the module's operational state. If any errors occur, such as an AT\_ERROR, they are immediately visible, allowing for quick troubleshooting.

Furthermore, the Serial Monitor logs responses received from the Sigfox module, such as the device information, mode, and firmware version. It confirms whether commands have been successfully processed and provides the Sigfox Device ID, which is essential for network registration and communication verification. The displayed OK responses indicate that the module is functioning correctly and executing commands as expected.

By analyzing these outputs, it is possible to ensure that the Sigfox module is properly configured, detect potential issues, and validate that the device is successfully communicating over the Sigfox network.

\begin{figure}[H]
	\centering
	\includegraphics[width=0.5\linewidth]{../../../Images/Syde/SigFox/SigFoxSerialMonitor}

In addition to displaying the Sigfox module’s initialization and device information, the Serial Monitor also provides feedback on the message transmission process (see Figure \ref{fig:SigFoxSerialMonitor}). When a message, such as "HELLO1234", is sent via Sigfox, the Serial Monitor logs each step of the transmission, ensuring that the data is correctly processed and transmitted over the 868 MHz frequency band.

The logs show the RF initialization (RF\_API\_init in TX), the starting transmission frequency, and the TX START command, which indicates the beginning of the message transmission. The number of bytes (nB=26) sent is also logged, followed by the waiting period until the transmission is complete. After transmission, the system waits for an acknowledgment (Wait For End of Tx), confirming that the message has been fully processed.

Once the transmission is finalized, the Serial Monitor displays responses such as "OnTxDone" and "TX END", signaling that the message has been successfully sent. The presence of multiple TX cycles in the logs suggests that the module follows the Sigfox protocol’s retransmission mechanism, ensuring message delivery reliability.

Finally, the Serial Monitor outputs an "OK" response, confirming that the Sigfox module is operating as expected. By analyzing these logs, it can be verified whether the Hypatia Board successfully sends message via SigFox.
