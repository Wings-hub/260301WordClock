\chapter{WiFi Connection Check}
When a client needs to verify the Wi-Fi connection of the Hypatia Board, a simple diagnostic program is required. The following code enables the ESP module on the board to establish a WiFi network and host a webpage. By accessing this webpage via the ESP's IP address, the user can remotely control an LED on the board. This functionality provides a straightforward way to confirm that the ESP is successfully connected and responsive

\section{Explanations}
\subsection{SSID}
The SSID (Service Set Identifier) is the name of a Wi-Fi network that allows devices to identify and connect to it. When scanning for available networks on a phone or computer, the list of visible names represents different SSIDs. Each wireless router or access point broadcasts an SSID to distinguish its network from others in the area.

\subsection{IP Address}
An IP address (Internet Protocol address) is a unique set of numbers assigned to a device connected to a network. It serves as an identifier, allowing devices to communicate with each other over the internet or within a local network.

The program starts by importing the necessary libraries \ref{lst:WiFi cpp}. WiFi.h enables the board to establish a network connection, while WebServer.h allows it to create and manage a simple web-based interface. The Adafruit NeoPixel library is included to control a WS2812B LED, and an additional image.h file contains graphical data, likely a logo that will be displayed on the web interface.

Next, the code defines key hardware and network settings. The LED is connected to pin 21, and the board is configured to join a predefined Wi-Fi network using a specific SSID and password. Once the connection is established, a web server is started on port 80, allowing access to the interface.

The board processes incoming HTTP requests, which correspond to specific actions. The code defines functions that change the LED’s color based on the request received. For example, if a request is sent to /red, the LED will switch to red, while /green and /blue activate their respective colors. Each function updates the LED and returns a response message, confirming the change. The server continuously listens for new requests, ensuring real-time interaction.

The main program execution begins in the setup() function, where the board initializes its serial communication, connects to Wi-Fi, and starts the web server. Once running, the loop() function ensures that all incoming web-based commands are continuously processed.

\section{Connecting Hypatia with WiFi}
To begin setting up and programming the Hypatia Board, the Arduino IDE must first be installed on your computer. In addition, the Adafruit Library is required to ensure proper functionality and enable communication with specific components of the board. It is important to verify that all necessary configurations for the Hypatia Board have been correctly applied.

Once the configurations are in place and the correct Board and COM Port have been selected, the next step is to adapt the code to your Wi-Fi network by entering the SSID and password at the designated locations in the code. This ensures that the Hypatia Board can successfully connect to the correct network and establish communication. Make sure to replace the placeholders with your actual network credentials to enable a stable connection. Afterwards compile and upload the provided code to the ESP32 on the Hypatia Board. This can be done by clicking the upload button (see Figure \ref{fig:ArduinoUpload}) located in the upper left corner of the Arduino IDE.
