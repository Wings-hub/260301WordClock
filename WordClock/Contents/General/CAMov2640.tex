\section{General}

Ein Kamera-Modul für einen Arduino ist ein Zusatzmodul, das an einen Arduino-Board angeschlossen werden kann, um Kamera-Funktionen zu ermöglichen. Hier sind einige allgemeine Merkmale eines Kamera-Moduls für Arduino:

\begin{description}
  \item [Bildsensor:] Ein Kamera-Modul enthält einen Bildsensor, der die Lichtintensität in der Umgebung misst und die Informationen in ein digitales Bild umwandelt.
  \item [Anschlüsse:] Ein Kamera-Modul hat typischerweise Anschlüsse für die Verbindung mit dem Arduino-Board, wie z.B. I\textsuperscript{2}C, SPI oder USB.
  \item [Bildformat:] Ein Kamera-Modul kann verschiedene Bildformate unterstützen, wie z.B. JPEG, RAW oder BMP.
  \item [Auflösung:] Ein Kamera-Modul kann eine bestimmte Auflösung bieten, wie z.B. 640 $\times$ 480 Pixel oder 1280 $\times$ 720 Pixel.
  \item [Bildrate:] Ein Kamera-Modul kann eine bestimmte Bildrate bieten, wie z.B. 30 Bilder pro Sekunde oder 60 Bilder pro Sekunde.
  \item [Steuerung:] Ein Kamera-Modul kann durch den Arduino gesteuert werden, um die Kamera-Funktionen zu kontrollieren, wie z.B. die Aufnahme von Bildern oder Videos.
  \item [Kommunikation:] Ein Kamera-Modul kann mit dem Arduino kommunizieren, um die Kamera-Funktionen zu steuern und die Bilder zu übertragen.
\end{description}

\bigskip

Einige Beispiele für Kamera-Module für Arduino sind:

  \begin{description}
    \item[Arducam:] Ein hochauflösendes Kamera-Modul mit einer Auflösung von 2 Megapixeln und einer Bildrate von 30 Bildern pro Sekunde. \Mynote{citation}
    \item [Raspberry Pi Camera:] Ein Kamera-Modul für den Raspberry Pi, das eine Auflösung von 5 Megapixeln und eine Bildrate von 30 Bildern pro Sekunde bietet.  \Mynote{citation}
    \item [OV7670:] Ein Kamera-Modul mit einer Auflösung von 640x480 Pixeln und einer Bildrate von 30 Bildern pro Sekunde.  \Mynote{citation}
  \end{description}


Es ist wichtig zu beachten, dass die spezifischen Merkmale eines Kamera-Moduls für Arduino je nach Modell und Hersteller variieren können.

\section{Library \FILE{Arducam}}

\subsection{Description}

Die Arducam-M-2MP unterstützt die Bibliothek \FILE{Arducam} für Arduino. Diese Bibliothek bietet eine einfache Möglichkeit, die Kamera-Funktionen des Arducam-M-2MP zu steuern und Bilder zu übertragen.

Die Basisbibliotheken bestehen aus zwei Unterbibliotheken: \FILE{ArduCAM} und \FILE{UTFT4ArduCAM\_SPI}. Diese beiden Bibliotheken sollten direkt unter die Bibliotheken des Arduino-Verzeichnisses kopiert werden, damit sie von der Arduino-IDE erkannt werden.

Die ArduCAM-Bibliothek ist die Kernbibliothek für ArduCAM-Shields. Sie enthält unterstützte Bildsensortreiber und Benutzerland-API-Funktionen, die Befehle zum Erfassen oder Lesen von Bilddaten erteilen. Es gibt auch ein Beispielverzeichnis innerhalb der ArduCAM-Bibliothek, das die meisten Funktionen der ArduCAM-Shields illustriert. Die vorhandenen Beispiele sind Plug-and-Play, ohne dass eine einzige Zeile Code geschrieben werden muss.

Die Bibliothek \FILE{UTFT4ArduCAM\_SPI} ist eine modifizierte Version von UTFT, die von Henning Karlsen geschrieben wurde. Wir haben sie portiert, um das ArduCAM-Shield mit LCD-Bildschirm zu unterstützen. Daher wird die Bibliothek \FILE{UTFT4ArduCAM\_SPI} nur benötigt, wenn das ArduCAM-LF-Modell verwendet wird.


Die Arducam-Bibliothek bietet folgende Funktionen:

\begin{itemize}
  \item Steuerung der Kamera-Funktionen, wie z.B. die Aufnahme von Bildern oder Videos
    \begin{itemize}
       \item   Beispiel \PYTHON{captureImage()}: Fängt ein Bild ein und speichert es auf dem Arduino.
       \item   Beispiel \PYTHON{startVideoRecording()}: Beginnt ein Video zu speichern.
       \item Beispiel \PYTHON{stopVideoRecording()}: Beendet das Video-Speichern.
     \end{itemize}
  \item Übertragung von Bildern über den USB-Anschluss
    \begin{itemize}
      \item  Beispiel \PYTHON{transferImage()}: Überträgt ein Bild vom Arduino zum Computer.
      \item   Beispiel: \PYTHON{transferVideo()}: Überträgt ein Video vom Arduino zum Computer.
    \end{itemize}
  \item Unterstützung für verschiedene Bildformate (JPEG, RAW)
    \begin{itemize}
      \item Beispiel \PYTHON{saveImageAsJPEG()}: Speichert ein Bild als JPEG-Datei.
      \item Beispiel \PYTHON{saveImageAsRAW()}: Speichert ein Bild als RAW-Datei.
    \end{itemize}
  \item Einfache Steuerung durch Arduino
    \begin{itemize}
      \item Beispiel \PYTHON{setCameraMode()}: Setzt den Kameramodus auf ``Bild'' oder ``Video''.
      \item Beispiel \PYTHON{setCameraResolution()}: Setzt die Auflösung der Kamera auf eine bestimmte Größe.
    \end{itemize}
\end{itemize}

\subsection{How to use}

Die Bibliotheken sollten vor dem Ausführen von Beispielen konfiguriert werden, andernfalls erhalten Sie eine Fehlermeldung beim Kompilieren.

\subsubsection{1. Edit \FILE{memorysaver.h} file}

Öffnen Sie die Datei \FILE{memorysaver.h} im ArduCAM-Ordner und aktivieren Sie die Hardwareplattform und das Kameramodul, das zu Ihrer Hardware passt, indem Sie die Makrodefinition in der Datei auskommentieren oder auskommentieren. Wenn Sie zum Beispiel eine ArduCAM-Mini-2MP haben, sollten Sie die Zeile \PYTHON{\#define OV2640\_MINI\_2MP} auskommentieren und alle anderen Zeilen auskommentieren. Und wenn Sie ein ArduCAM-Shield-V2 und ein OV5642-Kameramodul haben, sollten Sie die Zeile \PYTHON{\#define ARDUCAM\_SHIELD\_V2} und die Zeile \PYTHON{\#define OV5642\_CAM} auskommentieren und dann alle anderen Zeilen.

\subsubsection{2. Choose correct CS pin for your camera}

Öffnen Sie eines der Beispiele und verdrahten Sie die SPI- und I2C-Schnittstelle, insbesondere die CS-Pins, entsprechend den Beispielen mit dem ArduCAM-Shield. Hardware und Software sollten konsistent sein, um die Beispiele korrekt auszuführen.

\subsubsection{3. Upload the examples}


Im Beispielordner befinden sich sieben Unterverzeichnisse für verschiedene ArduCAM-Modelle und die Host-Anwendung. Der Ordner Mini ist für die Module ArduCAM-Mini-2MP und ArduCAM-Mini-5MP.

\begin{enumerate}
    \item Der Ordner \FILE{Mini\_5MP\_Plus} ist für ArduCAM-Mini-5MP-Plus (OV5640/OV5642) Module.
    \item Der Ordner \FILE{RevC} ist für ArduCAM-Shield-RevC oder ArduCAM-Shield-RevC+ Shields.
    \item Der Ordner \FILE{Shield\_V2} ist für das ArduCAM-Shield-V2 Schild.
    \item Der Ordner \FILE{host\_app} ist die Host-Erfassungs- und Anzeigeanwendung für alle ArduCAM-Module.
    \item Der Ordner \FILE{RaspberryPi} ist eine Beispielanwendung für die Raspberry Pi-Plattform, siehe weitere Anleitung.
    \item Der Ordner \FILE{ESP8266} ist für ArduCAM-ESP8266-UNO-Board-Beispiele für Bibliothekskompatibilität. Bitte versuchen Sie stattdessen, ESP8266 mit dem Skript josn board manager zu repositoryen.
\end{enumerate}
